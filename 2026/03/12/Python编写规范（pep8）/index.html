计算机基础

缩进

  • 不要使用tap缩进
  • 不要混用tap和空格

空格

  • list，dict，tuple，set后面加一个空格
  • dict的:后加一个空格
  • 注释符号#后加一个空格
  • 操作符两端加一个空格
  • 参数列表的=两端不加空格
  • 括号内的两端不需要空格
  • 不要在列表索引、函数调用、关键字参数赋值两侧使用空格

空行

  • function和class前后各用两个空行
  • class内部method之间用一个空白行分隔
  • 函数内逻辑无关的段落间空一行
  • if/for/while语句中即使执行语句只有一行也要另起一行

换行

  • 每行代码控制在80字符以内
  • 使用\和()控制换行

命名

  • 使用蛇形命名法
  • package/module名中不要出现下划线_
  • 函数、变量以及属性使用小写
  • 类和异常信息首字母大写，使用大驼峰命名
  • 模块级别的常量全部使用大写
类型 命名约定 例子
Class 每个单词都大写字母开头；不要用下划线 (PascalCase) Model, MyClass
Constant 使用大写字母；下划线分隔单词 (UPPER_CASE) CONSTANT, MY_CONSTANT, MY_LONG_CONSTANT
Package 使用小写单词；不用下划线分隔单词 package, mypackage
Function 使用小写字母；下划线分隔单词 (snake_case) function, my_function
Variable 使用小写字母；下划线分隔单词 (snake_case) x, var, my_variable
Method 使用小写字母；下划线分隔单词 (snake_case) class_method, method
Module 使用小写字母；下划线分隔单词 (snake_case) module.py, my_module.py

import

  • import之间进行分组，每组之间进行空行，组内尽量采用字典顺序排序，分组排序为标准库->第三方库->自主声明的文件
  • 引用一个模块式使用绝对的模块名称，例如要想引入bar包下面的foo模块，应该使用from bar import foo而不是import foo。
  • 必须使用相对引用时使用from . import foo（应该是用于与引入库区分）
  • 为了避免命名冲突，可以使用as或导入上一级命名空间

注释

  • 文档字符串docstring，是package，module，class，method，function级别的注释，可以通过__doc__成员访问到，注释在一对```之间
  • function，method的文档字符串应描述其功能、输入参数、返回值 ，如果有复杂的算法和实现，也需要写清楚
  • 不要写错误的和无谓的注释
  • 优先使用英文写注释，英文不好就全部用中文

异常

  • 不要轻易使用try/except
  • except后面需要指定捕捉的异常，裸露的except会捕捉所有异常，意味着会隐藏潜在的问题
  • 使用finally子句处理一些收尾操作，如关闭文件，关闭数据库连接等
  • try/except里的内容不要太多，只在可能抛出异常的地方使用
  • 从Exception而不是BaseException集成自定义的异常类 （这个没接触过，后面要学一下）

Class

  • 显示的写明父类，如果不是继承自别的类，就继承自object类（最基础的类对象）
  • 使用super调用父类的方法，这能明显的表明这是来自父类的方法
  • 支持多继承，即同时有多个父类，推荐使用Mixin(Mixin是指创建一些专门用于插入其他类中的插件类，这些类一般没有自己的属性，不会被用于实例化，主要用于扩展其他类的功能)

字符串

  • 使用字符串的join方法拼接字符串
  • 使用字符串类型的方法，而不是string模块的方法
  • 使用startswith和endwith方法比较前后缀
  • 使用f-string方法格式化字符串，即f"str1{content}str2"

比较

  • 空的list，str，tuple，set，dict和0.0，None都是False
  • 使用if somelist 而不是 if len(somelist) 判断为空
  • 使用is 和is not 与单例比较而不是== 和!=
  • 使用if a is not None而不是if not a is None，尽量看着复合人脑的处理规则
  • 使用isinstance()而不是type()进行类型判断
  • 使用in操作:用key in dict 而不是 dict.has_key()进行查询
  • 用set加速存在性检查，list的查找是线性的，set的底层是hash table（缺点是set需要使用更多内存空间）

其他

  • 使用内建函数sorted和.sort 进行排序
  • 适量使用map，reduce，filter和lambda，使用内建的all和any处理多个条件的判断
  • 使用with语句处理上下文
  • 利用好python的Duck Type特性
  • 使用logging记录日志，配置好格式和级别
  • 了解python的魔术方法
  • 阅读优秀开源代码
