ORM操作（此笔记为tortoise orm）

概述： ORM（对象关系映射）是一种编程技术，用于在关系数据库和面向对象编程语言之间建立桥梁。ORM 允许开发者以面向对象的方式操作数据库，而不必直接编写 SQL 查询语句。

一、数据库模型定义

Tortoise-ORM中的数据库模型是通过Python定义的，继承了tortoise.models.Model类的每个类代表数据库中的一张表，类的每个属性代表表中的一个列。需注意的是该类型类属于实例类，无法用于接口直接传回，只能在后端框架中使用。

基本模型定义

  1. 继承Model类。
  2. 定义字段。Tortoise-ORM提供了多种字段类定义模型的字段，如CharField，IntField、DatetimeField等。
  3. 字段类型约束。每个字段都需要指定数据类型。可添加对应类型的特定约束，如CharField类型的长度，DatatimeField类型的创建时生成时间戳，也可添加第三范式，如独一性，自增性，主键指定等。

常见字段类型

  1. CharField: 用于存储字符串，需要指定 max_length 参数，表示该字段最大允许的字符长度。
  2. IntField: 用于存储整数数据。
  3. FloatField: 用于存储浮动点数数据。适合表示小数值，例如价格、评分等。
  4. BooleanField: 用于存储布尔值（True 或 False）。
  5. TextField: 用于存储长文本数据，没有字符长度限制。适用于存储文章内容、评论等大量文本数据。
  6. DateField 和 DatetimeField: 分别用于存储日期和日期时间。DateField() 只存储日期，而 DatetimeField() 存储日期和时间。created_at=fields.DatetimeField(auto_now_add=True) 用于自动记录创建时间。
  7. ForeignKeyField: 用于定义模型间的一对多关系。例如，一个用户可以有多个订单，这时可以在 Order 模型中通过 ForeignKeyField 引用User模型。
  8. ManyToManyField: 用于定义多对多关系。例如，一个学生可以选修多门课程，而一门课程也可以有多个学生选修。
  9. JSONField: 用于存储 JSON 数据，适用于存储不确定结构的复杂数据。

字段约束：

  1. unique=True：独一性，该字段值不可重复。
  2. null=True：是否允许为空，sql默认字段不可填入空值。
  3. default=value：指定默认值。
  4. max_length=value：字段最大长度

Meta类约束

Model类中可定义Meta内部类，可用于定义模型的一些元信息，如表名、索引、约束等。 常见Meta类属性：

  • table：制定指定表名
  • indexes：定义索引字段，被添加索引属性的字段的查询操作性能显著高于未定义的字段。
  • unique_together：定义多个字段的联合唯一约束，确保多个字段的组合值是唯一的。
  • ordering：定义模型的默认排序规则，可指定用于排序的字段，如token的create_time属性。

二、异步与同步属性

概述： Tortoise-ORM中所有与数据库交互的操作都可以通过async/await语法进行异步处理。下为tortoise的函数表：

1.创建数据

方法 描述 示例代码 返回值
.create() 创建并保存单个对象 user = await User.create(name="Alice", age=20) 模型实例 (已保存)
.save() 保存实例 (新增或更新) user.name = "Bob"; await user.save() None
.bulk_create() 批量插入 (高性能) await User.bulk_create([u1, u2, u3]) None
.get_or_create() 获取或创建 (原子操作) obj, created = await User.get_or_create(name="Alice", defaults={"age": 20}) (实例, bool)
.update_or_create() 更新或创建 obj, created = await User.update_or_create(id=1, defaults={"age": 21}) (实例, bool)

2.查询数据

方法 描述 示例代码 异常处理
.get() 获取唯一匹配项 user = await User.get(id=1) 若无或多于1个，抛出 DoesNotExist 或 MultipleObjectsReturned
.first() 获取查询集的第一个 user = await User.filter(age=20).first() 若无返回 None
.last() 获取查询集的最后一个 user = await User.all().order_by("id").last() 若无返回 None

后为多对象操作
.all() 获取所有记录 users = await User.all() List[Model]
.filter() 条件过滤 (最常用) users = await User.filter(age__gt=18, name__contains="A") List[Model]
.exclude() 排除条件 users = await User.exclude(is_active=False) List[Model]
.limit() 限制数量 users = await User.all().limit(5) List[Model]
.offset() 偏移量 (分页) users = await User.all().offset(10).limit(5) List[Model]
.values() 返回字典列表 (非实例) data = await User.filter(id=1).values("id", "name") List[Dict]
.values_list() 返回元组列表 data = await User.all().values_list("id", "name") List[Tuple]
.count() 计数 count = await User.filter(age=20).count() int
.exists() 是否存在 (性能优于 count) if await User.filter(name="Alice").exists(): ... bool

3.更新数据

方法 描述 示例代码 返回值
.update() 批量更新 (直接生成 SQL UPDATE) rows = await User.filter(age=20).update(age=21) 受影响的行数 (int)
.save() 实例级更新 (先查后改) user.age = 21; await user.save(update_fields=["age"]) None
.increment() 原子自增 (F 表达式替代) await User.filter(id=1).increment(view_count=1) None (部分版本支持)

4.删除数据

方法 描述 示例代码 返回值
.delete() 批量删除 rows = await User.filter(age=20).delete() 受影响的行数 (int)
.delete() 实例级删除 await user.delete() None (或受影响行数)

5.高级查询条件

操作符 含义 示例 SQL 等价
exact 精确匹配 (默认) name="Alice" =
in 在列表中 id__in=[1, 2, 3] IN (...)
gt / gte 大于 / 大于等于 age__gt=18 > / >=
lt / lte 小于 / 小于等于 age__lt=18 < / <=
contains 包含 (模糊查询) name__contains="li" LIKE '%li%'
startswith 以...开头 name__startswith="A" LIKE 'A%'
endswith 以...结尾 name__endswith="e" LIKE '%e'
isnull 是否为 NULL email__isnull=True IS NULL
year / month 日期提取 created_at__year=2026 YEAR(...)
order_by 排序 .order_by("-age", "name") ORDER BY (-为降序)

6.关联查询

操作 描述 示例代码
正向查询 获取关联对象 company = await user.company (自动 lazy load)
company = await user.company.get() (显式)
反向查询 获取关联集合 users = await company.users.all()
预加载 (解决 N+1) 非常重要 users = await User.all().select_related("company")
companies = await Company.all().prefetch_related("users")
通过关联过滤 跨表查询 await User.filter(company__name="Google")

**补充：

  1. ** tortoise支持链式查询和条件查询，前者可进行查询语句套用，后者可在查询时根据字段进行条件判断。
  • exact：精确匹配
  • gte：大于等于
  • lte：小于等于
  • in：匹配指定列表中的任意值
  • isnull：检查字段是否为 NULL
  1. 排序与分页：Tortoise-ORM 支持通过 order_by() 方法进行排序，并且可以通过 limit() 和 offset() 实现分页功能。

三、数据库链接

初始化：

  1. Tortoise.init()
  2. register_tortoise()

关闭： Tortoise.close_connections()

aerich

一种orm迁移工具，需要结合tortoise使用。在register_tortoise的moduls参数部分（迁移类文件）添加aerich.models，然后便可使用aerich指令在终端进行创建、更新、查阅等操作，主要做应为可使用该工具对数据库进行回滚、统一读入读出等操作，对于开发安全性有益。
